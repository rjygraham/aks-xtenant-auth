# Orchestration Log: Bishop Session 2026-04-29T20:18:31

## Context

Bishop (Azure Engineer) fixed the cross-tenant blob write issue in the timestampwriter pod.

## Work Summary

Fixed cross-tenant Workload Identity by adding `AZURE_TENANT_ID` override to ConfigMap and deployment manifests.

### Root Cause

The AKS workload identity webhook always injects `AZURE_TENANT_ID` as the source (home) tenant. For cross-tenant scenarios, the Azure SDK uses this to determine token scoping. Since the target storage account is in a different tenant, Azure Storage rejected tokens scoped to the source tenant with `InvalidAuthenticationInfo: Issuer validation failed`.

### Solution

Override `AZURE_TENANT_ID` at the container level:
- Added `AZURE_TENANT_ID: "c8c22f13-9f4d-41a5-b619-eda5a1d50c39"` to `deploy/configmap.yaml`
- Added explicit `env` entry in `deploy/deployment.yaml` sourcing from ConfigMap (before webhook injection)

The webhook respects pre-existing env vars, so the ConfigMap value wins.
