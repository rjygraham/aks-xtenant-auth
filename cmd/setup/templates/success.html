Storage account details saved for tenant {{.TenantID}}

Complete the two steps below to finish deployment.

Step 1 — Assign the Storage Blob Data Contributor role

Run this command in Azure CLI:

az role assignment create \ --assignee-object-id $(az ad sp show --id {{.ClientID}} --query id -o tsv) \ --role "Storage Blob Data Contributor" \ --scope {{.ResourceID}} \ --assignee-principal-type ServicePrincipal

Run this command while logged into the target tenant ({{.TenantID}}) with an account that has Owner or User Access Administrator on the storage account. The timestamp writer will automatically begin writing once the role assignment propagates.

Step 2 — Update the AKS workload identity annotations

Edit deploy/serviceaccount.yaml with your app registration details:

azure.workload.identity/client-id: "{{.ClientID}}" azure.workload.identity/tenant-id: "{SOURCE_TENANT_ID}"

azure.workload.identity/tenant-id is the source tenant where the app registration lives — not {{.TenantID}}.
The timestamp writer reads storage account configuration directly from the shared database — no changes to deploy/configmap.yaml are required.
