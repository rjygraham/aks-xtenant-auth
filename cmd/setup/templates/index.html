Cross-Tenant Storage Access

Admin consent wizard for external tenant administrators

This wizard registers the AKS workload identity application in your Azure tenant, allowing a Kubernetes workload running in a separate tenant to write blobs to a Storage account you control.

Who should run this: a Global Administrator or Application Administrator in the target tenant.

Clicking the button below will redirect you to Microsoft's admin consent page. After you approve, you will be returned here with next steps — including the az role assignment create command to grant the service principal access to your storage account.
